## 📖 设计模式目录

### 创建型模式 (Creational Patterns)
创建型模式关注对象的创建机制。

| 模式 | 说明 | 应用场景 |
|------|------|---------|
| [单例模式](singleton/) | 确保类只有一个实例 | 配置管理、数据库连接池 |
| [工厂模式](factory/) | 定义创建对象的接口 | 对象创建逻辑复杂时 |
| [建造者模式](builder/) | 分步骤构建复杂对象 | 配置对象、复杂实体 |
| [原型模式](prototype/) | 通过克隆创建对象 | 对象创建成本高时 |

### 结构型模式 (Structural Patterns)
结构型模式关注类和对象的组合。

| 模式 | 说明 | 应用场景 |
|------|------|---------|
| [适配器模式](adapter/) | 转换接口使其兼容 | 第三方库集成 |
| [装饰器模式](decorator/) | 动态添加功能 | 中间件、功能增强 |
| [代理模式](proxy/) | 控制对象访问 | 延迟加载、缓存 |
| [外观模式](facade/) | 简化复杂系统接口 | 子系统封装 |

### 行为型模式 (Behavioral Patterns)
行为型模式关注对象之间的通信。

| 模式 | 说明 | 应用场景 |
|------|------|---------|
| [策略模式](strategy/) | 算法可以互相替换 | 支付方式、排序算法 |
| [观察者模式](observer/) | 一对多依赖关系 | 事件系统、消息通知 |
| [责任链模式](chain/) | 多个对象处理请求 | 审批流程、中间件 |
| [模板方法模式](template/) | 定义算法骨架 | 数据处理流程 |
| [命令模式](command/) | 将请求封装为对象 | 撤销/重做操作 |

## 🎯 学习路径

### 初学者（推荐顺序）
1. **单例模式** → 理解单例概念
2. **工厂模式** → 掌握对象创建
3. **策略模式** → 理解算法封装
4. **观察者模式** → 理解事件通知

### 进阶开发者
1. **建造者模式** → 复杂对象构建
2. **装饰器模式** → 动态功能扩展
3. **适配器模式** → 接口转换
4. **代理模式** → 访问控制

### 高级开发者
1. **责任链模式** → 请求处理链
2. **命令模式** → 请求封装
3. **模板方法模式** → 算法骨架
4. **外观模式** → 子系统封装

## 💡 实际应用场景

### Web 开发
- **中间件**：责任链模式、装饰器模式
- **路由**：策略模式
- **数据库连接**：单例模式
- **API 客户端**：适配器模式

### 微服务
- **服务发现**：观察者模式
- **配置管理**：单例模式
- **请求处理**：责任链模式
- **服务代理**：代理模式

### 系统设计
- **日志系统**：单例模式、装饰器模式
- **缓存系统**：代理模式
- **消息队列**：观察者模式
- **工作流引擎**：责任链模式、命令模式

## 🌟 Go 语言特色

本项目充分利用 Go 语言特性：

- **接口优先**：使用隐式接口实现，更灵活
- **组合优于继承**：使用嵌入而非继承
- **函数式特性**：函数作为一等公民
- **并发支持**：Channel 和 Goroutine
