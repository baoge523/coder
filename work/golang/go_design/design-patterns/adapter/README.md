# 适配器模式 (Adapter Pattern)

## 定义
将一个类的接口转换成客户希望的另外一个接口。适配器模式使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。

## 别名
包装器模式 (Wrapper Pattern)

## 分类
- 类适配器（使用继承）
- 对象适配器（使用组合）

## 组成部分
- Target: 目标接口
- Adaptee: 被适配者
- Adapter: 适配器

## 使用场景
- 系统需要使用现有的类，但接口不符合需求
- 想要建立一个可以重复使用的类
- 需要统一多个类的接口
- 第三方库接口适配

## 优点
- 提高类的复用性
- 增加类的透明性
- 灵活性好

## 缺点
- 过多使用会使系统凌乱
- 增加系统复杂度

## 示例说明
本示例实现了一个电源适配器，将不同电压转换为设备需要的电压。
