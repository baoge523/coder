# 建造者模式 (Builder Pattern)

## 定义
将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。

## 组成部分
- Product: 产品类
- Builder: 抽象建造者
- ConcreteBuilder: 具体建造者
- Director: 指挥者（可选）

## 使用场景
- 创建复杂对象
- 对象有多个组成部分
- 需要生成不同表示的对象
- 构建过程需要多个步骤

## 优点
- 封装性好
- 易于扩展
- 便于控制构建过程
- 支持链式调用

## 缺点
- 产生多余的 Builder 对象
- 内部结构复杂时，建造者也会变复杂

## Go 语言特点
Go 语言中常用链式调用实现建造者模式，代码简洁优雅。

## 示例说明
本示例实现了一个电脑配置的建造者，支持构建不同配置的电脑。
