# 责任链模式 (Chain of Responsibility Pattern)

## 定义
使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。将这些对象连成一条链，并沿着这条链传递该请求，直到有一个对象处理它为止。

## 组成部分
- Handler: 抽象处理器
- ConcreteHandler: 具体处理器
- Client: 客户端

## 使用场景
- 多个对象可以处理同一请求
- 不明确指定接收者
- 动态指定处理者集合
- 审批流程、日志处理、异常处理

## 优点
- 降低耦合度
- 增强灵活性
- 简化对象连接
- 责任分担

## 缺点
- 不保证请求一定被处理
- 性能影响
- 调试不便

## Go 语言实现特点
Go 语言中常用于实现中间件链和过滤器链。

## 示例说明
本示例实现了一个请假审批系统，不同级别的请求由不同级别的领导审批。
