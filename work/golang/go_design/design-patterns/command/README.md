# 命令模式 (Command Pattern)

## 定义
将一个请求封装为一个对象，从而使你可用不同的请求对客户进行参数化；对请求排队或记录请求日志，以及支持可撤销的操作。

## 组成部分
- Command: 命令接口
- ConcreteCommand: 具体命令
- Receiver: 接收者
- Invoker: 调用者
- Client: 客户端

## 使用场景
- 需要将请求调用者和请求接收者解耦
- 需要支持撤销操作
- 需要支持事务操作
- 需要记录日志
- GUI 按钮、菜单项、宏命令

## 优点
- 降低系统耦合度
- 新命令容易加入
- 容易实现命令队列和宏命令
- 容易实现撤销和恢复

## 缺点
- 可能产生大量具体命令类
