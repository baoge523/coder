# 装饰器模式 (Decorator Pattern)

## 定义
动态地给一个对象添加一些额外的职责。就增加功能来说，装饰器模式相比生成子类更为灵活。

## 组成部分
- Component: 抽象组件
- ConcreteComponent: 具体组件
- Decorator: 抽象装饰器
- ConcreteDecorator: 具体装饰器

## 使用场景
- 需要动态添加功能
- 需要动态撤销功能
- 不能使用继承扩展功能
- Java I/O 流、HTTP 中间件

## 优点
- 比继承更灵活
- 可以动态组合
- 符合开闭原则
- 可以使用多个装饰器

## 缺点
- 产生很多小对象
- 多层装饰比较复杂

## Go 语言实现特点
Go 语言中常用于实现中间件链。
