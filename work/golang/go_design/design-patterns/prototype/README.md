# 原型模式 (Prototype Pattern)

## 定义
用原型实例指定创建对象的种类，并且通过拷贝这些原型创建新的对象。

## 组成部分
- Prototype: 原型接口
- ConcretePrototype: 具体原型

## 使用场景
- 对象创建成本较大
- 需要大量相似对象
- 避免使用工厂层次结构
- 对象状态较少改变

## 克隆方式
- 浅拷贝：拷贝对象的值类型字段，引用类型字段仍指向原对象
- 深拷贝：完全拷贝对象及其引用的所有对象

## 优点
- 性能优良（避免重复初始化）
- 简化对象创建
- 可以动态增加或减少产品类

## 缺点
- 需要为每个类配备克隆方法
- 深拷贝实现较复杂

## Go 语言实现特点
Go 语言没有内置的克隆方法，需要手动实现。
