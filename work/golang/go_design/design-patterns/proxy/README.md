# 代理模式 (Proxy Pattern)

## 定义
为其他对象提供一种代理以控制对这个对象的访问。

## 代理类型
- 远程代理：为远程对象提供代理
- 虚拟代理：延迟创建开销大的对象
- 保护代理：控制对原始对象的访问权限
- 智能引用：在访问对象时执行额外操作

## 组成部分
- Subject: 抽象主题
- RealSubject: 真实主题
- Proxy: 代理

## 使用场景
- 延迟加载（虚拟代理）
- 访问控制（保护代理）
- 远程调用（远程代理）
- 日志记录
- 缓存代理

## 优点
- 职责清晰
- 高扩展性
- 智能化

## 缺点
- 增加系统复杂度
- 可能影响性能

## 与装饰器模式的区别
- 代理模式：控制访问
- 装饰器模式：增强功能
