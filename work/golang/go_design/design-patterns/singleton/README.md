# 单例模式 (Singleton Pattern)

## 定义
单例模式确保一个类只有一个实例，并提供一个全局访问点。

## 使用场景
- 数据库连接池
- 配置管理器
- 日志记录器
- 缓存管理

## Go 语言实现方式

### 1. 懒汉式（使用 sync.Once）
最推荐的 Go 语言实现方式，线程安全且高效。

### 2. 饿汉式
在包初始化时创建实例。

## 优点
- 控制实例数量
- 减少内存开销
- 提供全局访问点

## 缺点
- 违反单一职责原则
- 可能隐藏不良设计
- 在多线程环境下需要特殊处理

## 示例说明
本示例实现了一个配置管理器单例，展示了两种实现方式。
