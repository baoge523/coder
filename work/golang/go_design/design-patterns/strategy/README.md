# 策略模式 (Strategy Pattern)

## 定义
定义一系列算法，把它们一个个封装起来，并且使它们可以相互替换。策略模式让算法独立于使用它的客户而变化。

## 组成部分
- Strategy: 策略接口
- ConcreteStrategy: 具体策略实现
- Context: 上下文，持有策略引用

## 使用场景
- 多个类只有算法或行为不同
- 需要动态切换算法
- 避免使用多重条件判断
- 支付方式、排序算法、压缩算法等

## 优点
- 算法可以自由切换
- 避免多重条件判断
- 扩展性良好
- 符合开闭原则

## 缺点
- 策略类数量增多
- 客户端必须知道所有策略

## 示例说明
本示例实现了一个支付系统，支持多种支付方式的动态切换。
